## 📝 提交规范

建议采用以下提交信息格式：
<type>: <message>

例如：
feat: 添加hello.md测试文件
fix: 修复文件名拼写错误
doc: 更新README内容


常用 `type`:
- `feat`: 新功能
- `fix`: 修复问题
- `doc`: 文档更新
- `refactor`: 代码重构
- `test`: 添加测试
- `chore`: 其他杂项

---

## 📌 注意事项

- **提交前建议先 pull 最新主分支，避免冲突**。
- 保持良好的文档与提交记录习惯。
