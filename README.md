## 📚 学习任务建议

- [ ] 创建自己的文件夹，提交一个 `hello.md` 文件
- [ ] 提交规范测试（使用标准 commit message，如 feat / fix / doc 等）
- [ ] 创建并切换分支，提交新文件
- [ ] Pull Request 流程测试
- [ ] 模拟冲突并解决
- [ ] 合并代码并清理分支

---

## 📝 提交规范

建议采用以下提交信息格式：
<type>: <message>

例如：
feat: 添加hello.md测试文件
fix: 修复文件名拼写错误
doc: 更新README内容


常用 `type`:
- `feat`: 新功能
- `fix`: 修复问题
- `doc`: 文档更新
- `refactor`: 代码重构
- `test`: 添加测试
- `chore`: 其他杂项

---

## 📌 注意事项

- **请勿直接推送到主分支**（main/master），练习请走 PR 流程。
- **提交前建议先 pull 最新主分支，避免冲突**。
- 每人请在自己的子文件夹中练习，避免文件冲突。
- 保持良好的文档与提交记录习惯。
